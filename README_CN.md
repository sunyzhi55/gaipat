这是一份对仓库中英文说明文件的完整中文翻译与整理说明，覆盖以下两份原始文档的全部核心内容：

为了便于阅读，我把原始内容按主题重新组织，并在每一部分后补充了简短说明。原始文件内容不做修改；本文件只是一个新的中文导读版本。

## 1. 项目名称与目标

GAIPAT 的全称是 “Dataset on Human Gaze and Actions for Intent Prediction in Assembly Tasks”，可理解为“用于装配任务意图预测的人类注视与动作数据集”。项目使用 LEGO Duplo 积木搭建简单图形，并记录参与者在装配过程中的眼动、瞳孔信息、操作动作和任务状态。

这个数据集的核心目标是研究人类在装配任务中的视觉行为，并尝试预测操作者下一步会做什么，应用场景指向人机协作、工业辅助和意图预测。

原文对应位置：README.md 第 1 到 11 行；datasheet.md 第 17 到 19 行。

## 2. 数据集是如何采集的

### 2.1 实验目标

研究者希望比较不同眼动测量设备在装配任务中的意图预测效果。整个实验采用 LEGO Duplo 作为模拟工业装配场景的载体，任务形式是按照屏幕上的说明一步一步搭建图形。

说明：这意味着数据集并不只是“眼动数据”，而是一个带有明确任务背景的多模态数据集。每个样本都和装配步骤、屏幕说明以及实际动作相对应。

### 2.2 实验环境与材料

实验台由两部分组成：

- 一个装配工作台，中央是绿色装配区，两侧是积木存放区；
- 一个说明屏幕，用来显示当前装配图形和每一步的操作指令。

积木按颜色和形状组织，包括蓝色、红色、黄色、绿色，以及立方体和砖块两种形状。

参与者既有坐姿也有站姿，桌面高度会按欧洲人体工学标准 ISO-14738 调整。

说明：这种设置模拟了工业或协作装配环境，既考虑了空间布局，也考虑了姿势差异。

### 2.3 两种眼动采集配置

数据集包含两种采集方式：

1. 远程式配置

   使用 Fovio FX3 和 Tobii 4C。

   - Fovio FX3 负责跟踪工作台上的眼动。
   - Tobii 4C 负责跟踪说明屏上的眼动。
   - Fovio 的数据通过 EyeWorks 软件采集。
   - Tobii 4C 数据通过定制 Python 脚本采集。

   这种配置的特点是设备不戴在头上，而是放在环境中不同位置。

2. 头戴式配置

   使用 Pupil Labs Core 眼动眼镜。

   - 设备带有一个场景相机和两个红外相机；
   - 使用 AprilTag tag36h11 标记把工作区映射到场景相机；
   - 使用 Pupil Labs Capture 做场景映射、校准和采集。

说明：这两种配置分别代表“远程式眼动追踪”和“头戴式眼动追踪”，方便比较不同硬件条件下的数据质量和可用性。

原文对应位置：README.md 第 15 到 32 行；datasheet.md 第 165 到 174 行。

### 2.4 参与者

研究招募了 80 名心理学专业学生作为志愿者，分为四组，每组 20 人，按以下维度组合：

- 远程式 / 头戴式
- 坐姿 / 站姿

研究没有收集年龄、性别、健康状况等个人信息。参与者以学分或经验积分作为补偿。纳入条件是：能读懂法语说明，并且能够实际搬动积木。

说明：数据集的参与者信息是匿名化和最小化采集的，避免了直接身份识别风险。

原文对应位置：README.md 第 34 到 36 行；datasheet.md 第 176 到 182 行。

### 2.5 实验流程

每位参与者的实验流程基本一致：

1. 先校准并验证眼动设备。如果校准质量不够好，就重复校准。若完全无法校准，实验仍继续，但对应数据不记录。
2. 第一个说明会显示在屏幕上，告诉参与者要搭建什么图形。实验员先放好第一块积木，其余积木由参与者完成。
3. 每一步屏幕都会显示视觉指令，指出要拿哪一块、放到哪里，并用箭头强调移动方向。每个图形需要 6 到 12 步完成。参与者通过桌下按钮切换到下一步。

整个过程每个图形约 5 分钟，6 个图形总共约 30 分钟；包含接待、同意书和讲解后，每人总时长约 50 分钟。

说明：这说明数据是严格按“步骤”组织的，而不是随意连续记录的自然行为日志。

原文对应位置：README.md 第 38 到 51 行；datasheet.md 第 158 到 163 行和第 171 到 174 行。

### 2.6 采集软件与硬件规格

采集软件包括：

- Pupil Capture 和 Pupil Player，版本 3.5，用于头戴式眼动采集；
- EyeWorks，版本 3.30，用于远程式采集。

实验时段是 2023 到 2024 年冬季。

设备与环境尺寸如下：

- Dell P2416D 屏幕：52.7 x 29.6 cm，2560 x 1440 像素；
- Plexiglass 支撑桌：100 x 50 cm；
- LEGO 装配表面：76 x 38 cm。

说明：这些尺寸对于理解坐标系、AOI 区域和后续视觉对齐非常重要。

原文对应位置：README.md 第 53 到 66 行；datasheet.md 第 180 到 182 行。

### 2.7 演示视频说明

README 提供了一个演示视频，展示使用头戴式眼动仪采集数据的过程。需要注意：

- 视频中的人是实验员，不是正式参与者；
- 实验员演示的是 Car 图形；
- 画面包含两种摄像角度；
- 眼动通过设备中的红外相机记录；
- 录像中只保留与实验相关的内容，不记录参与者脸部、眼睛或身体的完整视频，以确保匿名性。

说明：这个视频主要是帮助理解采集流程，而不是数据本身。

## 4. setup 目录里的文件是什么意思

### 4.1 blocks.csv

这个文件描述积木：

- id：积木编号，范围是 0 到 23；
- color：颜色，包含 blue、red、green、yellow；
- shape：形状，包含 cube 和 brick。

说明：如果你想把数据和具体积木对应起来，这个文件是基础索引表。

原文对应位置：README.md 第 113 到 116 行。

### 4.2 instructions_{figure_name}.csv

这个文件描述每个图形的步骤说明：

- id：指令编号，0 到 n；其中第 0 步由实验员完成；
- block：要移动的积木编号；
- origin：移动前该积木的四个角坐标和层级；
- destination：移动后该积木的四个角坐标和层级。

这些位置是相对坐标。（相对于table坐标系）

说明：这类文件定义的是“任务逻辑”，也就是每一步应该移动哪块积木、从哪里移到哪里。

原文对应位置：README.md 第 118 到 124 行。

### 4.3 slides_{figure_name}.csv

这个文件描述屏幕上每一页的 AOI 区域：

- id：幻灯片编号；
- title：顶部说明区域；
- block：要抓取的积木区域；
- destination：目标放置区域；
- figure：整张图形相关区域；
- arrow：箭头区域。

这些区域同样是相对坐标。（相对于screen坐标系）

说明：这部分是做屏幕 AOI 分析时最关键的元数据。

原文对应位置：README.md 第 126 到 134 行。

### 4.4 participants.csv

这个文件描述参与者和录制状态：

- id：参与者编号；
- setup：采集配置，remote 或 head mounted；
- position：姿势，sitting 或 standing；
- pupil / fovio / tobii：校准质量，取值可能是 impossible、severe issues、slight issues、no issue；如果不适用则记为 NA；
- figure_name：该图形是否被录制，1 表示录到，0 表示没录到。

说明：这个表可以帮助你快速判断某个参与者、某个设备、某个任务是否可用。

原文对应位置：README.md 第 136 到 141 行。

### 4.5 glasses.csv

这个文件记录佩戴眼镜与否的分布：

- setup：采集配置；
- position：坐姿或站姿；
- glasses：戴眼镜的人数；
- no_glasses：不戴眼镜的人数。

说明：它常用于分析眼镜是否会影响眼动质量。

原文对应位置：README.md 第 143 到 147 行。

## 5. participants 目录里的文件是什么意思

每个参与者目录下，会按图形拆分成多个子目录，例如 car、house、sc、tb、tc、tsb。每个图形下面又分 table 和 screen 两个视角。

### 5.1 table 子目录

table 下有：

- gazepoints.csv：工作台上的注视点；（相对于table坐标系）
- events.csv：工作台上的事件；
- states.csv：工作台状态；
- pupil_info.csv：瞳孔大小和置信度。

### 5.2 screen 子目录

screen 下有：

- gazepoints.csv：屏幕上的注视点；（相对于screen坐标系）
- events.csv：屏幕上的事件，主要是按钮操作；
- states.csv：屏幕状态；
- pupil_info.csv：瞳孔大小和置信度。

说明：一个参与者在一个任务上的记录，实际上包含两个同步视角：看桌面和看屏幕。

原文对应位置：README.md 第 152 到 164 行；datasheet.md 第 86 到 98 行。

## 6. 各个数据文件的字段含义

### 6.1 gazepoints.csv

字段包括：

- timestamp：时间戳；
- x, y：注视点坐标。

补充说明：

- 不同眼动仪的数据已经统一标准化；
- 只保留了双眼 gaze 点，因为不是所有设备都能分别提供左右眼数据；
- 如果某些设备只提供左右眼分开的数据，研究者按照厂商建议计算了双眼 gaze 点；
- 经过设备文档要求做了过滤；
- 没有数据的位置记为 NaN。

说明：这说明数据在入库前已经做过跨设备对齐，适合统一分析，但也意味着它不是原始未加工输出。

原文对应位置：README.md 第 168 到 172 行。

### 6.2 pupil_info.csv

字段包括：

- timestamp：时间戳；
- right_diameter、left_diameter：左右眼瞳孔直径；
- confidence：置信度指标。

补充说明：

- 不同眼动仪的 confidence 不可直接比较；
- 头戴式数据只提供一个全局置信度；
- 远程式数据提供左右眼的置信度。

说明：做跨设备建模时，这个字段需要特别小心处理，不能直接拿来做横向数值比较。

原文对应位置：README.md 第 174 到 178 行。

### 6.3 events.csv

工作台上的事件包括：

- start
- grasp
- release
- end

其中 block 字段表示被操作的积木编号。

屏幕上的事件包括：

- start
- next
- previous
- end

说明：

- 工作台事件对应真实物理动作；
- 屏幕事件对应任务导航操作；
- start 和 end 在两边是相同的。

原文对应位置：README.md 第 180 到 192 行。

### 6.4 states.csv

工作台上的状态包括：

- timestamp：时间戳；
- 每个积木的四个角坐标、level 和 held。

其中 held 表示该积木是否被参与者拿起。

屏幕上的状态包括：

- timestamp：时间戳；
- id：当前幻灯片编号。

说明：table 的 states 能帮助重建每一时刻工作台上积木的摆放；screen 的 states 能帮助知道当前屏幕显示的是哪一步。

原文对应位置：README.md 第 194 到 203 行。

## 7. 数据概况统计

根据 setup/participants.csv 的统计：

- 总参与者数：80；
- setup 分布：head_mounted 40 人，remote 40 人；
- position 分布：sitting 40 人，standing 40 人；
- 四个组合组别分别各 20 人：
  - head_mounted + sitting：20
  - head_mounted + standing：20
  - remote + sitting：20
  - remote + standing：20

各任务的录制覆盖数如下：

- car：64
- tb：65
- house：66
- sc：67
- tc：67
- tsb：66

对应的缺录条数分别是：

- car：16
- tb：15
- house：14
- sc：13
- tc：13
- tsb：14

另外，有 13 条记录标记为 impossible calibration，表示某些参与者在某些设备上无法成功校准，因此对应记录没有有效数据。

说明：这类统计很重要，因为它决定了后续分析时哪些样本可用、哪些样本需要剔除或单独处理。

## 8. 数据质量、噪声和缺失

数据集明确提到以下问题：

- 眼动追踪可能因为眨眼、设备偏移或校准误差而丢失数据；
- 不同参与者的操作策略不同，会带来行为差异；
- 某些录制根本无法完成，因此 setup/participants.csv 里会标出缺失。

说明：这意味着 GAIPAT 不是“完全干净”的实验数据，而是一个现实世界采集得到的、多噪声来源的数据集。

原文对应位置：datasheet.md 第 104 到 124 行。

## 9. 数据的匿名性与伦理

数据集不包含敏感信息，也没有收集可直接识别个人身份的人口统计信息。研究者声明：

- 参与者数据已匿名化；
- 不能从数据中重建个人身份；
- 伦理审查已通过 CERGA；
- 参与者同意数据收集与使用；
- 参与者保留了自己的唯一标识，并可请求删除数据。

说明：对于做人类行为数据分析的人来说，这部分是必须阅读的合规信息。

原文对应位置：datasheet.md 第 130 到 152 行；第 184 到 212 行。

## 10. 数据预处理与公开性

数据已经做过标准化处理。研究者还说明：

- 原始数据是公开的；
- 预处理脚本也是公开的；
- 使用的是自定义 Python 脚本。

说明：这对于复现实验和二次开发非常有用，因为你既能用现成数据，也能回到原始数据重新构建。

## 12. 你可以如何理解这个数据集

如果用一句话概括，GAIPAT 是一个“带任务语义的、多模态、双视角、按步骤组织的人类装配行为数据集”。它同时包含：

- 眼动轨迹；
- 瞳孔直径与置信度；
- 工作台上的物理动作；
- 屏幕上的任务步骤；
- 每一步对应的世界状态。

这使它特别适合做：

- 下一步动作预测；
- 注视行为分析；
- 眼动仪之间的比较；
- 人机协作中的意图识别；
- 任务步级对齐分析。

## 13. 阅读建议
